Installation Guide

Installing on a server/localhost

This is a guide to help you install the Maru application on a server/local development environment. The process involved is almost identical if not for the small changes that the environment itself requires to be done and not the application.

1: Copy File(s)

Once you have purchased Maru, and extracted the contents of the downloaded file then you will be able to find the zip file named dist.zip. This is the file that will be needed now.

To install on a server, just copy the dist.zip file on to the public hosting space of the server. This should mostly be the public_html directory of your server. Once you have copied, make sure to set the root directory of your domain to public_html/public directory. Making this setting is very important for the proper functioning of the application.

2: Create Database

Your hosting platform should have some form of a functionality to create a database. Use that wizard/functionality to create a new database. This has to be a new database and an empty one for proper installation and functioning of the application. Once the database is created, please make a note of the database's name somewhere safe as it will be needed down the line.

3: Create Database User

After a database is created, a database user with full previliges to the newly created database needs to be created, if one does not exists. Again, make a note of the database user's username and password for now as it will be required when installing the application.

4: Proceeding with the installer

That is all the tedious work needed! Now, open your browser and open your domain. The GUI installer will automatically open. Proceed with entering the required information as like explained further below and the installation will be done in a breeze.

Purchase Authentication

Provide the Envato username you used to purchase Maru and the purchase code you got along with the purchase. You can use this as a guide to get your purchase code.

Server Requirements

This is an automated check to make sure the server satisfies the requirements of the application and also has all the requires PHP/Apache extensions to function properly. If some error is shown, kindly fix the errors on the server side and refresh the page to make sure all options are checked to be present. All the extensions needs to be available to proceed to the next step.

Access Requirements

This is again an automated check to make sure the application has the appropriate read/write access to specific locations of the filesystem for porper functioning. If any errors are shown, make sure to fix them and refresh the page. All the checks needs to be passed for the installer to be able to proceed to the next step.

Database Details

This is where to enter the details of the database and the database user that you would have created and noted down previously. Some details on the form are pre-filled which are mostly not required to be changed. But if by any chances your environment needs a specific change of these settings, please make sure the correct values are entered in the form before proceeding.

NOTE: As of now, the application setup can only be supported by a relational database and the setup is made for a MySQL connection. So, it cannot be changed.

Super Admin Details

Every application needs an admin and its time to create one for ours too. Enter the email of the admin, and the required password. This password and the username of administrator will be used for the Super-Admin login to the application. So please use a secure and a rememberable password.

NOTE: The Super-Admin can only take the name as Administrator and the username administrator. Also, as of this release the super-admin's password cannot be changed. The provision to do that will be made available in future releases.

Installing

Once you have entered the data in the previous step, and clicked the INSTALL button, it is going to take considerable time to proceed to the next step. Please DO NOT click on the refresh or anywhere on the application window as this might make the installation faulty and the application might not work properly.

Finalising Installation

As soon as the previous step is completed, you will be moved to final step of the installtion wizard. This is again an automated step to make sure everything was done right. This should all be in green and nothing needs to be done. Just click on the button at the bottom to take you to login page and you can use the username administrator and the perviously entered password to make the first login into the application.

Restarting Installation

Before clicking the button to take you to your login page, if at any step of the installation wizard you want to restart the installation, enter the url, https://{your-domain}/installer/step/0 manually in the browser. Once you visit the login page from the installtion wizard's last step, you have to remove all the application files manually and redo the installation process.
